# 🎨 Image Generation using Variational Autoencoder (VAE)

A Deep Learning project that implements a Variational Autoencoder (VAE) using PyTorch for image reconstruction, image generation, and latent space exploration. The model learns compressed representations of images and generates new samples from the learned latent space.

## 📖 Overview

Generative Artificial Intelligence focuses on creating new content rather than simply analyzing existing data.

This project implements a Variational Autoencoder (VAE), a neural network architecture capable of learning meaningful latent representations of images. The model is trained on the CIFAR-10 dataset and demonstrates the ability to reconstruct images, generate entirely new images, and create smooth transitions between image representations.

The project provides a practical introduction to Generative AI, Computer Vision, and Representation Learning.

## 🧠 What is a Variational Autoencoder?

A Variational Autoencoder (VAE) is a generative neural network that learns a compressed representation of data while preserving important information.

Unlike traditional autoencoders, a VAE learns a probability distribution in latent space rather than a fixed representation. This allows the model to generate entirely new samples by randomly sampling from the learned distribution.

VAEs are widely used in:

- Image Generation
- Data Augmentation
- Feature Learning
- Representation Learning
- Generative AI Applications

## 🏗️ Model Architecture

The model consists of three major components:

### Encoder

The encoder receives an input image and compresses it into a lower-dimensional latent representation.

Responsibilities:

- Feature extraction
- Dimensionality reduction
- Learning latent variables

---

### Latent Space

The latent space stores compressed image information.

The encoder learns:

- Mean (μ)
- Log Variance (logσ²)

These values are used to generate latent vectors through a reparameterization process.

---

### Decoder

The decoder reconstructs images from latent vectors.

Responsibilities:

- Image reconstruction
- Image generation
- Latent space visualization

---

## ⚙️ How It Works

### Step 1: Dataset Loading

The CIFAR-10 dataset is loaded and preprocessed.

Dataset Characteristics:

- 60,000 Color Images
- 10 Classes
- Resolution: 32 × 32 Pixels

---

### Step 2: Encoding

Input images are passed through multiple convolutional layers.

The encoder transforms images into:

- Mean Vector (μ)
- Log Variance Vector (logσ²)

These vectors describe the learned latent distribution.

---

### Step 3: Reparameterization

The latent vector is generated using:

```text
z = μ + σ × ε
```

Where:

- μ = Mean
- σ = Standard Deviation
- ε = Random Noise

This technique enables backpropagation while maintaining stochastic sampling.

---

### Step 4: Decoding

The decoder receives the latent vector and reconstructs the original image.

The reconstructed image is then compared with the input image.

---

### Step 5: Loss Calculation

The model uses two loss components:

### Reconstruction Loss

Measures how accurately the reconstructed image matches the original image.

### KL Divergence Loss

Encourages the latent space to follow a normal distribution.

Total Loss:

```text
Total Loss = Reconstruction Loss + KL Divergence
```

---

### Step 6: Training

The model is trained using:

- Adam Optimizer
- CIFAR-10 Dataset
- Multiple Training Epochs

The objective is to minimize reconstruction errors while learning meaningful latent representations.
